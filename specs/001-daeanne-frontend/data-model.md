# Data Model: Daeanne Frontend Web App

**Date**: 2026-06-11

## Entities

### Task (read from Dispatcher — no new storage)

The frontend reads task data from the existing Dispatcher via the Bridge relay. The `AgentTask` model in `Daeanne.Shared` is the source of truth. The frontend uses a projected DTO (`TaskDto`) that includes only fields needed for display.

| Field         | Type      | Source                            | Notes                                              |
| ------------- | --------- | --------------------------------- | -------------------------------------------------- |
| Id            | int       | AgentTask.Id                      | Primary key                                        |
| Type          | string    | AgentTask.Type (enum→string)      | e.g., "Research", "Email", "TrendAnalyzer"         |
| Topic         | string    | Derived from AgentTask.Prompt     | First ~80 chars of the prompt, truncated           |
| Status        | string    | AgentTask.Status (enum→string)    | Running, Pending, Blocked, Succeeded, Failed, etc. |
| Age           | string    | Computed from AgentTask.CreatedAt | Human-readable ("2m ago", "1h ago")                |
| CreatedAt     | DateTime  | AgentTask.CreatedAt               | ISO 8601                                           |
| CompletedAt   | DateTime? | AgentTask.CompletedAt             | ISO 8601; null if not complete                     |
| ResultJson    | string?   | AgentTask.ResultJson              | Full output for task detail view                   |
| Error         | string?   | AgentTask.Error                   | Error message for failed tasks                     |
| CorrelationId | string?   | AgentTask.CorrelationId           | Links commands to results                          |

**Filter rules**: The task list API returns:

- All tasks with status Running, Pending, Blocked, Awaiting
- Succeeded/Failed/TimedOut/Escalated tasks where CompletedAt is within last 24 hours
- Ordered by: active tasks first (by CreatedAt desc), then completed tasks (by CompletedAt desc)

### Command (new — frontend-originated)

A command is a free-text instruction sent from the chat interface. It flows through Service Bus to the Dispatcher.

| Field         | Type   | Notes                                                    |
| ------------- | ------ | -------------------------------------------------------- |
| Prompt        | string | Free-text command from Jeffrey                           |
| CorrelationId | string | UUID generated by the API Function; used to track result |
| TaskType      | string | Default "Generic"; may be overridden in future           |

Serialized as `FrontendRequest` record in `Daeanne.Shared.Models`.

### Response (new — Dispatcher result for a command)

| Field         | Type    | Notes                                   |
| ------------- | ------- | --------------------------------------- |
| CorrelationId | string  | Matches the command's correlationId     |
| Succeeded     | bool    | Whether the task completed successfully |
| Response      | string? | Full response content                   |
| Error         | string? | Error message if failed                 |

Serialized as `FrontendResult` record in `Daeanne.Shared.Models`.

### Trend Highlight (read from Dispatcher)

Trend data is surfaced from completed TrendAnalyzer tasks. The API Function queries recent completed tasks of type `TrendAnalyzer` and extracts highlights.

| Field      | Type           | Notes                                       |
| ---------- | -------------- | ------------------------------------------- |
| Date       | DateOnly       | The date the trend was analyzed             |
| Highlights | List\<string\> | Extracted highlight strings from ResultJson |

> **Note**: The API response wraps this entity as `{ date, highlights[] }` — see contracts/api.md `GET /api/trends/today`.

### Chat Message (client-side only — not persisted)

The chat view displays a conversational history. Messages are held in the Blazor client's memory during the session. They are not persisted — reloading the app starts fresh.

| Field         | Type     | Notes                                                    |
| ------------- | -------- | -------------------------------------------------------- |
| Id            | string   | UUID                                                     |
| Direction     | enum     | Sent (Jeffrey → Daeanne) or Received (Daeanne → Jeffrey) |
| Content       | string   | Message text                                             |
| Timestamp     | DateTime | When sent/received                                       |
| Status        | enum     | Sending, Sent, Delivered, Error                          |
| CorrelationId | string?  | Links sent messages to their responses                   |

## State Transitions

### Command Lifecycle

```
[User types command]
    → ChatMessage(Direction=Sent, Status=Sending)
    → POST /api/command → 202 Accepted + correlationId
    → ChatMessage(Status=Sent)
    → Poll GET /api/result/{correlationId}
    → Response arrives
    → ChatMessage(Direction=Received, Status=Delivered)
```

If POST fails: `ChatMessage(Status=Error)`, input text preserved.
If polling times out (>5 min): `ChatMessage(Direction=Received, Content="Request may not have been processed", Status=Error)`.

### Task Status Display Mapping

| Dispatcher Status           | Display Group   | Badge Color |
| --------------------------- | --------------- | ----------- |
| Running                     | Running         | Blue        |
| Pending                     | Pending         | Amber       |
| Blocked, Awaiting           | Blocked         | Red         |
| Succeeded, Partial          | Succeeded (24h) | Green       |
| Failed, TimedOut, Escalated | Failed (24h)    | Red         |
| Deferred, Future            | Hidden          | —           |

## Relationships

- A **Command** creates an **AgentTask** on the Dispatcher (via Bridge relay)
- A **Command** has exactly one **Response** (identified by correlationId)
- A **Task** has zero or one **full output** (ResultJson, available after completion)
- A **Trend Highlight** is derived from a completed **Task** of type TrendAnalyzer
- **Chat Messages** are linked to Commands/Responses by correlationId
